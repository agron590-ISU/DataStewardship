Slides about making slides

Rmarkdown

  • Great for reproducibility
  • Very nice for including figures, tables, data, and code
  • Strong choice for header -> bullet point type style
  • Output html which enables easy use of life-changing GIFs

Rmarkdown

  • Not great for creating diagrams and schematics while writing your presentation
  • Appearance is difficult to change unless you can write CSS/HTML
    • However, CSS templates are out there
    • Some HTML relatively easy to find and implement e.g.:
      <center><img src="images/awake.gif" width="400px"/></center>

Three Rmarkdown options in RStudio

  • ioslides (html)
  • Slidy (html)
  • Beamer (pdf)

*ioslides has more default styling than Slidy, but both can be changed using a styling sheet (CSS)

YAML Options

  • Rendering – incremental: true
  • Screen size –widescreen: true
  • Text size – smaller: true
  • Transiton speed – transition: slower
  • Custom CSS – css: styles.css
